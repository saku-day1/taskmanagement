# タスク管理アプリケーション

カンバンボード形式でタスクの進捗を管理するWebアプリケーション。AIコース課題として、要件定義・設計・実装の一連のプロセスを体験する目的で開発。

---

## 機能

- **タスク作成** — タイトル（必須）・説明文・期限日・優先度を入力して追加
- **カンバン表示** — 未着手 / 進行中 / 完了 の3列でタスクをカード表示
- **ステータス移動** — ← / → ボタンで隣の列へ移動
- **タスク編集** — 編集モーダルで各項目を更新
- **タスク削除** — 確認ダイアログ付きで削除
- **期限切れ表示** — 期限超過のタスクは赤文字＋「期限切れ」バッジを表示
- **優先度ラベル** — 高（赤）/ 中（黄）/ 低（緑）で色分け表示

## セットアップ・起動手順

### 前提条件

- Docker Desktop がインストールされていること
- JDK 21 がインストールされていること
- Node.js がインストールされていること

### 起動順序（厳守）

```
① DB（Docker） → ② バックエンド → ③ フロントエンド
```

#### 1. DB 起動

```bash
docker compose up -d
```

#### 2. バックエンド起動

```bash
cd backend
./gradlew bootRun
```

#### 3. フロントエンド起動

```bash
cd frontend
npm install
npm run dev
```

### アクセス先

| サービス | URL |
|---------|-----|
| フロントエンド | http://localhost:5173 |
| バックエンドAPI | http://localhost:8080 |
| DB | localhost:5433 |

## データモデル

```
TASK
├── id          : 自動採番（主キー）
├── title       : タイトル（必須・最大50文字）
├── description : 説明文（任意・最大200文字）
├── deadline    : 期限日（任意・YYYY-MM-DD）
├── priority    : 優先度（high / medium / low）
├── status      : ステータス（todo / in_progress / done）
├── created_at  : 作成日時
└── updated_at  : 更新日時
```
