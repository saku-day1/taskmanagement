フロントエンドのESLintとバックエンドのCheckstyleを実行し、品質チェックの結果をまとめてください。

## 実行手順

### ① フロントエンド ESLint
- ディレクトリ: `frontend/`
- コマンド: `npm run lint`
- 確認対象: `src/` 以下の TypeScript / TSX ファイル全体

### ② バックエンド Checkstyle
- ディレクトリ: `backend/`
- コマンド: `./gradlew checkstyleMain`
- 確認対象: `src/main/java/` 以下の Java ファイル全体

## 結果の報告形式

両コマンドを順に実行し、完了後に以下の形式で日本語でまとめてください。

```
## 品質チェック結果

### フロントエンド (ESLint)
- エラー: N 件
- 警告: M 件
[問題があれば] ファイル名:行番号 - 内容

### バックエンド (Checkstyle)
- 違反: N 件
[問題があれば] ファイル名:行番号 - ルール名: 内容

### 総評
問題なし / 要対応: X 件
```

問題が見つかった場合は、修正方法を提案してください。
