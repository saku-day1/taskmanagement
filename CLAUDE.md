## Git / GitHub 運用ルール

### 基本方針
- **mainへの直接プッシュは禁止**（GitHub側でブランチ保護を設定済み）
- すべての作業はイシュー起票 → ブランチ作成 → PR の流れで行う

### イシュー運用
- 作業を始める前に必ずGitHubイシューを作成する
- タイトル: 日本語で簡潔に（例: `タスク一覧APIの実装`）
- ラベルを付ける: `feature`（新機能）/ `bug`（不具合）/ `docs`（ドキュメント）/ `chore`（雑務）

### ブランチ命名規則
```
<type>/issue-<番号>-<短い説明>
```
| type | 用途 |
|------|------|
| `feature` | 新機能 |
| `fix` | バグ修正 |
| `docs` | ドキュメント |
| `chore` | 設定・雑務 |

例: `feature/issue-1-task-list-api`、`docs/issue-3-update-readme`

### PRルール
- タイトル: `[#イシュー番号] 日本語の説明`（例: `[#1] タスク一覧APIの実装`）
- 本文に `Closes #番号` を書いてイシューと紐付ける
- mainへのマージはPR経由のみ（直接プッシュ不可）

### コミットメッセージ
- 日本語で記述する
- 例: `タスク一覧エンドポイントを追加`

### 危険な操作
- `rm` / `git reset --hard` / `git clean` / `git push --force` 等は必ず実行前にユーザーに確認する

## サーバー起動ルール

### 起動順序（厳守）
```
① DB（Docker） → ② バックエンド → ③ フロントエンド
```
DBが起動していない状態でバックエンドを起動すると、PostgreSQL接続エラーで落ちる。

### 各サービス

| 対象 | ディレクトリ | コマンド | URL |
|------|------------|---------|-----|
| DB（PostgreSQL） | プロジェクトルート | `docker compose up -d` | localhost:5433 |
| バックエンド | `backend/` | `./gradlew bootRun` | http://localhost:8080 |
| フロントエンド | `frontend/` | `npm run dev` | http://localhost:5173 |

### ポート競合ルール
- ポートが競合した場合は既存プロセスを**必ず停止**してから起動する
- 別ポートでの代替起動は禁止（フロント↔バックエンド間の接続が壊れるため）
- カスタムコマンド `/start` で詳細手順を確認できる
